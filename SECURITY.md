# Security And Data Handling

This repo is designed for public learning with fictional data.

## Do Not Add

- real customer data
- private company notes
- personal information
- raw confidential transcripts
- API keys
- access tokens
- passwords
- confidential business information

## Safe Data Types

Use one of these:

| Data Type | Safe To Use? | Notes |
| --- | --- | --- |
| Fictional data | Yes | Best for public examples |
| Synthetic data | Yes | Useful for demos and tests |
| Fully anonymized data | Maybe | Use only when approved by your company |
| Real customer data | No | Keep it out of this public sandbox |

## AI Usage

Use approved AI tools only. Before sending any note to an AI assistant, remove personal information, company-sensitive details, and contractual or commercial information.

AI output should be reviewed against the source notes before it influences product decisions.

## Local Files

The `.private/` folder is ignored by Git. Use it only for local working notes, task lists, or private planning context.
