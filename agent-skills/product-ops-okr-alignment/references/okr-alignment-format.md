# OKR Alignment Format

Use the schema in `ai-workflows/schemas/okr_alignment.schema.json`.

Each alignment should include:

| Field | Meaning |
| --- | --- |
| objective | Product outcome the work may support |
| key_result | Measurable result tied to the objective |
| linked_metric | Metric used to track movement |
| supporting_evidence | Evidence from notes, metrics, opportunity map, or roadmap score |
| related_opportunity | Opportunity that may support the outcome |
| roadmap_candidate | Candidate initiative connected to the key result |
| alignment_assessment | Strong, partial, or weak |
| human_review_needed | Decision or evidence gap for a product leader |

The output should make alignment visible. It should not approve OKRs.
